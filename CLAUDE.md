# 요구사항 정의서
## 오늘의 섬김 — 구글 폼 연동 기능

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 오늘의 섬김 (셀 역할 추첨 + 셀 리더 일지 연동) |
| 작성일 | 2026년 3월 25일 |
| 대상 독자 | 개발자 |
| 문서 버전 | v1.0 |

---

## 1. 프로젝트 개요

교회 소그룹(셀) 모임에서 매주 반복되는 두 가지 작업을 하나의 앱으로 처리한다.

1. **역할 추첨** — 참석자 이름을 입력하면 랜덤으로 역할을 배정한다.
2. **셀 리더 일지 제출** — 추첨 결과 및 나눔 기록을 구글 폼에 자동으로 채워 넘긴다.

---

## 2. 사용자

| 구분 | 설명 |
|------|------|
| 주 사용자 | 셀 리더 (매주 모임을 진행하는 담당자) |
| 사용 환경 | 모바일 브라우저 (iOS / Android) |
| 기술 수준 | 일반 사용자 (비개발자) |

## 3. 기능 요구사항

### 3-1. 역할 추첨

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|----------|
| F-01 | 참석자 이름 입력 | 이름 추가 / 삭제, Enter 키 지원 | 필수 |
| F-02 | 역할 자동 배정 | 인원 수에 따라 역할 목록 자동 결정 | 필수 |
| F-03 | 셔플 애니메이션 | 추첨 시 2초간 이름 섞이는 효과 표시 | 선택 |
| F-04 | 결과 카드 표시 | 역할별 이름 카드를 순차적으로 표시 | 필수 |
| F-05 | 다시 추첨 | 결과 초기화 후 재추첨 가능 | 필수 |
| F-05-1 | 추첨 없이 넘어가기 | 추첨 생략하고 나눔 기록 입력 단계로 바로 이동 | 필수 |

**역할 배정 규칙**

| 인원 | 배정 역할 |
|------|-----------|
| 4명 | 첫기도, 끝기도, 기도정리, 질문자 |
| 5명 | + 사진사 |
| 6명 | + 인도 |
| 7명 이상 | 나머지 인원 "통과" 처리 |

### 3-2. 구글 폼 연동

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|----------|
| F-06 | 폼 링크 생성 | 추첨 결과 및 나눔 기록을 URL 파라미터로 변환 | 필수 |
| F-07 | 폼 자동 채우기 | 출석 멤버(1번), 나눔 기록(3번) 항목에 데이터 자동 입력 | 필수 |
| F-08 | 폼 열기 | 새 탭 또는 외부 브라우저로 폼 오픈 | 필수 |
| F-09 | 링크 복사 | 생성된 폼 URL을 클립보드에 복사 | 권장 |
| F-10 | 나눔 기록 입력 단계 | 참석자별 나눔 내용을 개별 입력 칸으로 제공 | 필수 |
| F-11 | 나눔 기록 포맷 변환 | 입력된 내용을 `이름- 내용` 형식으로 자동 변환하여 폼 3번에 삽입 | 필수 |
| F-12 | 나눔 기록 임시 저장 | 입력 내용을 로컬에 저장하여 나중에 다시 불러올 수 있음 | 필수 |


### 3-3. 앱 화면 흐름 (Flow)

```
[1단계] 이름 입력
    ↓
[2단계-A] 추첨 시작  →  결과 확인
[2단계-B] 추첨 없이 넘어가기 (버튼)
    ↓
[3단계] 나눔 기록 입력
    - 참석자별 개별 입력 칸 제공
    - 예: 홍길동 [          ]
          김철수 [          ]
    - 나중에 다시 돌아와 수정 가능 (임시 저장)
    ↓
[4단계] 셀 리더 일지 열기
    - 출석 멤버(1번) + 나눔 기록(3번) 자동 채워진 폼 오픈
    - 링크 복사 버튼 제공
```

#### 나눔 기록 포맷 변환 예시

**입력**

| 이름 | 나눔 내용 |
|------|-----------|
| 홍길동 | 말씀이 도전이 됐다 |
| 김철수 | 기도가 필요한 한 주였다 |
| 이영희 | (빈칸 — 생략) |

**entry.924151839 에 삽입되는 값**
```
홍길동- 말씀이 도전이 됐다
김철수- 기도가 필요한 한 주였다
```

> ⚠️ 빈칸인 멤버는 생략. 줄바꿈은 `%0A`로 인코딩.

## 5. 제약사항 및 참고

- 구글 폼은 **공식 외부 API를 제공하지 않음** — URL 파라미터 방식(Pre-fill)만 가능
- Pre-fill 방식은 **폼을 열어주는 것**까지만 가능하며, **자동 제출은 불가**
- 사용자가 폼을 열고 나머지 항목 입력 후 직접 제출해야 함
- 완전 자동 제출이 필요할 경우 **Google Apps Script** 방식으로 별도 구현 필요
